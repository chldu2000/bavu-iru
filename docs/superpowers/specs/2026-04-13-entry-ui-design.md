# 密码条目基本 UI 设计

## 概述

为 Bavu-Iru 密码管理器实现密码条目的完整 CRUD UI，覆盖从应用启动到条目管理的全部交互流程。采用 Master-Detail 分栏布局，象牙色/暖橙配色方案。

## 布局

Master-Detail 分栏，占满窗口高度：

| 区域 | 宽度占比 | 内容 |
|------|---------|------|
| 左侧列表 | 35% | 搜索框 + 条目列表 + 新建按钮 |
| 右侧面板 | 65% | 根据状态切换：空状态引导 / 条目详情 / 编辑表单 |

最小窗口宽度 640px，分栏不可拖拽调整。

## 四个核心状态

### 状态 1：锁屏

应用启动时展示，居中布局：

- 应用标题 "Bavu-Iru" + 副标题 "密码管理器"
- 主密码输入框
- "解锁保险库" 按钮
- "创建新保险库" 文字链接（首次使用时）
- 已有保险库时只显示解锁流程；无保险库时自动进入创建流程

### 状态 2：空列表

解锁后无条目时展示，双面板空状态：

- 左侧：图标 + "还没有条目，点击下方按钮添加"
- 右侧：图标 + "选择一个条目查看详情，或创建新条目"
- 底部 "新建条目" 按钮始终可见

### 状态 3：条目详情（查看模式）

点击列表中条目后展示：

- 顶部：条目标题 + 操作按钮（编辑 / 删除）
- 字段展示（只读）：
  - 用户名 — 右侧 "复制" 按钮
  - 密码 — 默认隐藏 `••••••••••`，👁 切换 + "复制" 按钮
  - 网址 — 可点击链接样式
  - 备注 — 多行文本
- 点击 "编辑" 进入状态 4
- 点击 "删除" 弹出确认提示

### 状态 4：编辑 / 新建

编辑现有条目或创建新条目：

- 顶部： "编辑条目" 或 "新建条目" 标题
- 表单字段：
  - 标题（必填，输入框，聚焦时边框高亮为强调色）
  - 用户名（输入框）
  - 密码（默认隐藏 `••••••••••`，👁 切换 + "生成" 按钮）
  - 网址（输入框）
  - 备注（多行文本域）
- 底部： "取消" + "保存" 按钮
- 保存后返回查看模式

## 密码字段行为

查看和编辑模式统一：

- **默认隐藏**：显示为 `••••••••••`
- **👁 切换**：点击眼睛图标在明文/隐藏之间切换
- **编辑模式**：输入内容实时存储，但显示始终受 👁 控制
- **生成按钮**：点击后填入新密码，同样默认隐藏

## 交互流程

```
启动应用 → 锁屏（输入主密码 / 创建新保险库）
         → 解锁成功 → 主界面（空列表引导 / 加载条目列表）
                    → 点击条目 → 查看详情（复制 / 编辑 / 删除）
                              → 点击编辑 → 编辑模式（保存/取消）
                              → 点击删除 → 确认弹窗 → 移除
                    → 点击新建 → 空编辑表单 → 保存 → 加入列表
```

## 删除确认

点击删除后使用 `window.confirm()` 弹出确认："确定要删除 [条目标题] 吗？"确认后删除条目并清空右侧面板。

## 组件拆分

| 组件 | 职责 |
|------|------|
| `LockScreen.svelte` | 锁屏/创建保险库界面 |
| `EntryList.svelte` | 左侧条目列表 + 搜索 |
| `EntryDetail.svelte` | 右侧条目查看面板 |
| `EntryForm.svelte` | 条目编辑/新建表单 |
| `PasswordField.svelte` | 密码字段（隐藏/显示 + 复制） |
| `EmptyState.svelte` | 空状态引导展示 |

## 前端 Store 更新

### entries store

在现有 `entries` writable store 基础上，添加加载和 CRUD 操作：

- `loadEntries()` — 调用 `invoke('entry_list')` 获取全量条目
- `createEntry(entry)` — 调用 `invoke('entry_create', { entry })`
- `updateEntry(entry)` — 调用 `invoke('entry_update', { entry })`
- `deleteEntry(id)` — 调用 `invoke('entry_delete', { id })`

### vault store

现有 store 已有 unlock/lock/initialize 方法，需连接后端 invoke 调用。

### tauri.ts 工具函数

补充条目 CRUD 的 invoke 封装。
